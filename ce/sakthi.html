🚀 CivicEye: AI-Based Pothole Detection System

📌 Abstract

CivicEye is an AI-driven system that detects potholes from images and automatically generates complaints using geo-tagged data. The system reduces manual reporting and improves response time using deep learning and automation.

🎯 Objective

  • Detect potholes using AI
  • Capture location using GPS
  • Estimate severity, cost, and manpower
  • Automatically send complaint to authorities

⚙️ System Architecture (Mermaid)

graph TD

A[User Uploads Image] --> B[Flask Backend]

B --> C[YOLOv8 Model]
C --> D[Detect Pothole]

D --> E[Severity Calculation]
E --> F[Priority Assignment]
F --> G[Cost & Manpower Estimation]

B --> H[GPS Input (Lat, Lon)]

G --> I[Generate Complaint Template]

H --> I

I --> J[Send Email to Roads Department]

J --> K[Display Result to User]

🧠 Technologies Used

  • YOLOv8 (Deep Learning)
  • Flask (Backend)
  • OpenCV (Image Processing)
  • HTML, CSS (Frontend)
  • SMTP (Email Automation)

🔄 Workflow

  1. User uploads pothole image
  2. System captures GPS location
  3. AI detects pothole using YOLO
  4. Severity is calculated
  5. Priority is assigned
  6. Cost and manpower estimated
  7. Complaint is generated
  8. Email is sent to authority

📊 Output

  • Pothole Count
  • Severity Score
  • Priority Level
  • Estimated Cost
  • Manpower Required
  • Location Details

⚠️ Limitations

  • Currently supports only pothole detection
  • Depends on image quality
  • Requires GPS input

🚀 Future Scope

  • Multi-issue detection (garbage, water leak)
  • Mobile app integration
  • Government API integration
  • Real-time monitoring

🎤 Conclusion

CivicEye provides a smart and automated solution for civic issue reporting using AI and GPS integration. It improves efficiency, transparency, and response time in urban infrastructure management.
